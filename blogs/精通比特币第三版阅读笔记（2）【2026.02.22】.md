# 比特币交易的运作模式

- 比特币系统整体结构
  - 参与者
    - 用户：通过钱包持有密钥并控制资产
    - 矿工：通过竞争性计算生成区块
  - 核心对象
    - 交易：在网络中传播的价值转移指令
    - 区块链：所有交易的权威账簿
  - 工具
    - 区块链浏览器：用于查询地址、交易、区块关系
      - 可搜索：地址 / 交易哈希 / 区块高度 / 区块哈希
      - 隐私风险：查询行为可能与IP和身份关联
- 使用场景示例：在线商店支付
  - 商户生成包含信息的比特币URI二维码
    - 地址
    - 金额
    - 标签
    - 描述
  - 钱包扫描后自动填充支付信息
  - 用户签名并广播交易
- 比特币交易的本质
  - 定义
    - 所有者授权价值转移
    - 形成连续的所有权链
  - 结构
    - 输入（Inputs）：引用之前的未花费输出
    - 输出（Outputs）：定义新的所有权
    - 输入总额 ≥ 输出总额
      - 差额 = 交易费
  - 所有权证明
    - 通过数字签名完成
    - 节点可独立验证
- 交易链机制
  - 每个输入引用前一笔交易的输出
    - 通过 txid + 输出索引定位
  - 输入不直接包含金额
    - 需查找原交易确认价值
  - 交易形成连续链条
- 找零机制
  - 输入不可拆分
  - 通常包含：
    - 支付输出
    - 找零输出
  - 找零地址
    - 通常使用新地址
    - 提升隐私
  - 无找零交易
    - 仅在金额匹配时出现
- 币种选择（Coin Selection）
  - 钱包自动选择输入
    - 聚合多个小额UTXO
    - 使用单个较大UTXO
  - 目标
    - 平衡UTXO碎片化
    - 控制手续费
- 常见交易类型
  - 简单支付
    - 1 输入 + 2 输出（支付 + 找零）
  - 合并交易
    - 多输入 + 单输出
    - 用于整理UTXO
  - 批处理交易
    - 1 输入 + 多输出
    - 企业批量支付
- 构建交易流程
  - 钱包内部完成
    - 选择输入
    - 创建输出
    - 计算手续费
  - 可离线签名
    - 不需要实时联网
- 输入获取逻辑
  - 全节点
    - 维护完整UTXO集合
  - 轻钱包
    - 仅跟踪自身UTXO
  - 不足时合并多个UTXO
- 输出创建逻辑
  - 使用脚本锁定
    - “提供正确签名即可解锁”
  - 找零在同一交易中创建
  - 手续费
    - 隐含于输入输出差额
- 交易传播机制
  - P2P网络
    - 节点互联
  - Gossip协议
    - 接收到新交易后转发
  - 数秒内传播至全网
- 接收方视角
  - 钱包检测到相关输出
  - 验证交易结构与签名
  - 全节点可验证UTXO合法性
- 交易进入区块链
  - 被矿工纳入候选区块
  - 成功挖出区块即获得首次确认
  - 每增加一个新区块 = 增加一次确认
- 确认机制的安全含义
  - 回滚交易需重算替代区块
  - 选择累计工作量最高链
  - 六次确认通常视为高度安全
- 花费确认后的交易
  - 输出成为新的UTXO
  - 可继续被引用
  - 形成更长交易链

> 比特币交易整体逻辑结构可以抽象为：
>
> - 用户用密钥签名构建交易
> - 交易通过P2P网络传播
> - 矿工通过PoW将交易写入区块
> - 区块叠加形成不可逆历史
> - UTXO不断被消费与生成
> - 所有权通过交易链持续转移
